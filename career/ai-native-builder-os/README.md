## Product Thesis

### Chosen Personal Product Direction

Choose one main theme only:

**Personal efficiency × content understanding**

Work-side dev efficiency and testing quality are adjacent leverage, but not the main personal product battlefield.

### Product Entry Point

Move from **information saving** to **information judgment and understanding**.

Core value:
- Is this worth reading?
- What is the author's stance?
- What is fact vs assumption vs subjective claim?
- Is the argument balanced?
- What are the strongest supporting and opposing views?

### Product Form Suggestion

Start from a narrow wedge:
- browser extension for article quality / stance / structure judgment
- then deepen into richer understanding workflows

## Compliance / Feasibility Notes

- Must consider copyright, platform rules, content processing boundaries, and explicit user authorization.
- Current safest direction: operate in the user's real browser context and on content the user explicitly chooses to process, instead of building broad unauthorized scraping pipelines.
